進度欄。

基礎架構。

首先第一步我們先把這次要做的分層架構給準備好，一樣先開好一個Core Web應用程式，這次我們選擇空白專案。

建立完後我們有了初始化的結構。

接下來我們開始進行分層，這邊分層會拆成類別庫，至於是不是需要都拆成類別庫才能做分層模式呢？
答案是：不用的，小的專案可以用資料夾做分門別類就可以了。
而我們這邊示範就以類別庫的方式呈現。

在現在的解決方案點選右鍵 > 加入 > 新增專案 > 選擇類別庫。

這樣我們的Service層架構就建完了。
接下來把剩下Repository完成(備註：Common層這次示範不會用到就不特別建了)。

13、建立各層的AutoMapper。

Service層，我們目前需要對照的是Repository回傳的Model與Service要回傳給Contoller的Model，其中有兩個datetime的model我們需要做特殊處理轉換。

    public class ServiceProfile : Profile
    {
        public ServiceProfile()
        {
            this.CreateMap<AccountDataModel, AccountDto>()
                .ForMember(x => x.CreateDate, y => y.MapFrom(z => z.CreateDate.ToString("yyyy/MM/dd")))
                .ForMember(x => x.ModifyDate, y => y.MapFrom(z => z.ModifyDate.ToString("yyyy/MM/dd")));
        }
    }

可別忘了我們的規格。
a、僅可取得帳號、電話、信箱、建立日期、異動日期、異動人員。
b、個資規定使用者電話後四碼必須以＊字號取代。

所以我們需要把電話後四碼以＊字號取代，我們建立一個private function來做這件事。

        /// <summary>
        /// 轉換電話號碼
        /// </summary>
        /// <param name="phoneNumber"></param>
        /// <returns></returns>
        private string ConvertPhoneNumber(string phoneNumber)
        {
            if (string.IsNullOrWhiteSpace(phoneNumber))
            {
                return phoneNumber;
            }
            else
            {
                return (phoneNumber.Substring(0, 6) + new string('*', phoneNumber.Length - 6));
            }
        }

把這方法加入原本的function順便補上一些錯誤處理。

        /// <summary>
        /// 取得單筆帳號資訊
        /// </summary>
        /// <param name="account"></param>
        /// <returns></returns>
        public AccountDto GetAccount(string account)
        {
            if (string.IsNullOrWhiteSpace(account))
            {
                throw new Exception("Account 不可為空 !");
            }

            var data = this._accountRepository.GetAccount(account);
            var result = this._mapper.Map<AccountDto>(data);
            result.Phone = ConvertPhoneNumber(result.Phone);
            return result;
        }

開始實作。

        /// <summary>
        /// 取得單筆帳號資訊
        /// </summary>
        /// <param name="account"></param>
        /// <returns></returns>
        public AccountDataModel GetAccount(string account)
        {
            var sql = @"SELECT [Account]
                              ,[Password]
                              ,[Phone]
                              ,[Email]
                              ,[CreateDate]
                              ,[ModifyDate]
                              ,[ModifyUser]
                          FROM [Northwind].[dbo].[Users]
                          WHERE Account = @Account";

            var parameter = new DynamicParameters();
            parameter.Add("@Account", account, DbType.String);

            using (IDbConnection conn = this._databaseHelper.GetConnection())
            {
                var result = conn.QueryFirstOrDefault<AccountDataModel>(
                    sql,
                    parameter);

                return result;
            }
        }
