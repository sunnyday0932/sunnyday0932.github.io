開放封閉原則 Open-Close Principle(OCP)

定義

軟體設計原則，應該對擴展開放，對修改封閉。

甚麼意思呢？

  • 1、對擴展開放 - 程式碼對於新增功能是透過新增程式碼進行。
  • 2、對修改封閉 - 新增功能時不應該是更改現有程式碼。

那為甚麼要使用OCP？

  • 1、程式碼可維護性提高。
  • 2、程式碼可擴展性提高。
  • 3、程式碼可複用、靈活性高。
  • 4、程式碼可測試性提高。

我們來看一段例子，傳統汽車對於軟體硬體整合，還不夠靈活的時候，需要更改東西就需要把整台汽車拆開某些部分增加功能。

	 //擴充功能
        public class 擴充功能
        {
            public string 加入影音撥放器()
            {
                var 步驟 = new string[] { "拆開汽車", "放入影音撥放器" };

                if (步驟[0] == "拆開汽車")
                {
                    if (步驟[1] == "放入影音撥放器")
                    {
                        return "安裝影音撥放器成功";
                    }
                }
                return "安裝失敗";
            }

            public string 加入自動導航裝置()
            {
                var 步驟 = new string[] { "拆開汽車", "放入自動導航裝置" };

                if (步驟[0] == "拆開汽車")
                {
                    if (步驟[1] == "放入自動導航裝置")
                    {
                        return "安裝自動導航裝置成功";
                    }
                }
                return "安裝失敗";
            }
        
	 private static void Main(string[] args)
        {
            擴充功能 汽車 = new 擴充功能();
            Console.WriteLine(汽車.加入影音撥放器());
            Console.WriteLine(汽車.加入自動導航裝置());
        }

我們會發現每次要安裝新功能都需要分成兩個步驟，先拆開汽車、再安裝東西，這樣每次要對車子增加功能都是一項大工程；而特斯拉改良了這個方式，硬體已經再造車的時候準備好，要擴充功能只需要更新軟體即可，於是程式就變成了這個樣子…

	//將擴充功能做成介面
	public interface 擴充功能
        {
            public string 更新軟體();
        }
		
	//每個要安裝的新功能再透過繼承介面來實作
        public class 安裝影音播放功能 : 擴充功能
        {
            public string 更新軟體()
            {
                return "影音播放功能安裝成功";
            }

        }

        public class 安裝自動導航功能 : 擴充功能
        {
            public string 更新軟體()
            {
                return "自動導航功能安裝成功";
            }
        }
		
	private static void Main(string[] args)
        {
            安裝影音播放功能 ModelS = new 安裝影音播放功能();
            Console.WriteLine($"ModelS{ModelS.更新軟體()}");

            安裝自動導航功能 Model3 = new 安裝自動導航功能();
            Console.WriteLine($"Model3{Model3.更新軟體()}");
        }

如此一來每個功能需要安裝的時候就不需要把Tesla整台拆開，只需要更新軟體即可達到擴充功能；再更新功能的時候也不需要去修改原方法，已達到OCP(開放封閉)的精神。
