dotnet Core HangFire - 4(使用技巧與設定下篇)

1、Trigger(觸發程序)。

看到這裡，我一開始以為所謂的Trigger就只有類似於在SQL Server當觸動某張Table時，會觸發某種事件這樣，但其實並不只是只有此種才稱之為Trigger，所以一開始看到大神介紹此種方法的時候還一頭霧水。

我們可以將排程做成Trigger的方式，由程式啟動的時候觸發部屬排程工作。
順便會帶一點將排程執行的邏輯拆程類似之前分層架構的模式。

首先我們建立一個ITimeService介面、與實作就是取得當下時間。

    public interface ITimeService
    {
        public DateTime GetCurrentTime();
    }
    
    public class TimeService : ITimeService
    {
        public DateTime GetCurrentTime()
        {
            var currentTime = DateTime.Now;
            return currentTime;
        }
    }

接下來建立一個HangfireJob的介面與類別，這邊實體就會使用到剛剛所建立的TimeService。

    public interface IHangfireJob
    {
        void GetCurrentTime();
    }
    
    public class HangfireJob : IHangfireJob
    {
        private readonly ITimeService _timeService;

        public HangfireJob(
            ITimeService timeService)
        {
            this._timeService = timeService;
        }

        public void GetCurrentTime()
        {
            var startTime = this._timeService.GetCurrentTime();
            Console.WriteLine($"起始時間：{startTime.ToLocalTime():yyyy/MM/dd HH:mm:ss}");

            Thread.Sleep(1000);

            var endTime = this._timeService.GetCurrentTime();
            Console.WriteLine($"結束時間:{endTime.ToLocalTime():yyyy/MM/dd HH:mm:ss}");
        }
    }

接著建立HangfireJobTrigger，這邊程式一啟動會先把現有的job清除，然後將預設的Recurring job 加入排程。

    public interface IHangfireJobTrigger
    {
        void OnStart();
    }
    
    public void OnStart()
        {
            //清除所有的排程
            using var conn = JobStorage.Current.GetConnection();
            conn.GetRecurringJobs().ForEach(row =>
            {
                RecurringJob.RemoveIfExists(row.Id);
            });

            //Recurring job
            RecurringJob.AddOrUpdate<IHangfireJob>
                (
                    methodCall: x => x.GetCurrentTime(),
                    cronExpression: "0/5 * * * *",
                    timeZone: TimeZoneInfo.Local
                );
            
            //把原本在StartUp的Job也一併搬到這邊
            RecurringJob.AddOrUpdate
                (
                    recurringJobId: "Recurring job",
                    () => Console.WriteLine($"Recurring job觸發時間：{DateTime.Now}"),
                    cronExpression: "*/1 * * * *",
                    TimeZoneInfo.Local
                );

            RecurringJob.AddOrUpdate
                (
                    recurringJobId: "alpha job",
                    () => Console.WriteLine($"Recurring alpha job觸發時間：{DateTime.Now}"),
                    cronExpression: "*/1 * * * *",
                    TimeZoneInfo.Local,
                    queue: "alpha"
                );

            RecurringJob.AddOrUpdate
                (
                    recurringJobId: "beta job",
                    () => Console.WriteLine($"Recurring beta job觸發時間：{DateTime.Now}"),
                    cronExpression: "*/1 * * * *",
                    TimeZoneInfo.Local,
                    queue: "beta"
                );
        }

現在程式的架構。

接著我們把注入的Job注入的地方統一整理在一起。
建立一個DependencyInjectionExtension。

    public static class DependencyInjectionExtension
    {
        public static IServiceCollection AddDependencyInjection(this IServiceCollection services)
        {
            //Service
            services.AddScoped<ITimeService, TimeService>();

            //Jobs
            services.AddScoped<IHangfireJobTrigger, HangfireJobTrigger>();
            services.AddScoped<IHangfireJob, HangfireJob>();

            return services;
        }
    }

目前程式的架構。

最後一步我們回到StartUp改變一下Hangfire的注入。
第一在ConfigureServices要把剛剛DI Extension注入。

第二在Configure的地方需要做點改變。

 public void Configure(
 IApplicationBuilder app, 
 IWebHostEnvironment env, 
 IServiceProvider serviceProvider
 //多使用serviceProvider)
 {
     //觸發Job
    var hangfireTrigger = serviceProvider.GetService<IHangfireJobTrigger>();
    hangfireTrigger.OnStart();
 
 }

2、Hangfire.Console：在儀錶板觀看排程的log。

可以使用這種方式來記錄排程執行階段是否有遇到狀況。

首先需要安裝Hangfire.Console。

接著到StartUp的ConfigureService加入設定。

修改一下HangfireJob的方法。

    public interface IHangfireJob
    {
        void GetCurrentTime(PerformContext context);
    }
    
    public class HangfireJob : IHangfireJob
    {
        private readonly ITimeService _timeService;

        public HangfireJob(
            ITimeService timeService)
        {
            this._timeService = timeService;
        }

        public void GetCurrentTime(PerformContext context)
        {
            var startTime = this._timeService.GetCurrentTime();
            var startMessage = $"起始時間：{startTime.ToLocalTime():yyyy/MM/dd HH:mm:ss}";
            context.WriteLine(startMessage);

            Thread.Sleep(1000);

            var endTime = this._timeService.GetCurrentTime();
            var endMessage = $"結束時間:{endTime.ToLocalTime():yyyy/MM/dd HH:mm:ss}";
            context.WriteLine(endMessage);
        }
    }

Trigger的地方也要修改。

            //Recurring job
            RecurringJob.AddOrUpdate<IHangfireJob>
                (
                    methodCall: x => x.GetCurrentTime(null),
                    cronExpression: "0/5 * * * *",
                    timeZone: TimeZoneInfo.Local
                );
