dotnet Core HangFire - 3(使用技巧與設定上篇)

Written by Sian on April 3, 2021 in 學習筆記 觀看次數:

前言

本篇會介紹一些特殊使用技巧，跟一些可以調整的設定。

本文

1、API觸發排程。

首先建立一個API Controller。


    [Route("api/[controller]")]
    [ApiController]
    public class HangfireApiController : ControllerBase
    {
        [Route("enquee")]
        [HttpGet]
        public string TestHangfire()
        {
            BackgroundJob.Enqueue(
                () => Console.WriteLine("API觸發：Fire and Foreget Schedule!"));

            return "排程觸發";
        }
    }

2、設定排程Priority。

先在StartUp中ConfigureService設定Priority。

services.AddHangfireServer(options =>
            {
                //priority
                options.Queues = new[] { "alpha", "beta", "default" };
            });

將Recurring Job 改成：

            //Recurring job
            RecurringJob.AddOrUpdate
                (
                    recurringJobId: "Recurring job",
                    () => Console.WriteLine($"Recurring job觸發時間：{DateTime.Now}"),
                    cronExpression:"*/1 * * * *",
                    TimeZoneInfo.Local
                );

            RecurringJob.AddOrUpdate
                (
                    recurringJobId: "alpha job",
                    () => Console.WriteLine($"Recurring alpha job觸發時間：{DateTime.Now}"),
                    cronExpression: "*/1 * * * *",
                    TimeZoneInfo.Local,
                    queue: "alpha"
                );

            RecurringJob.AddOrUpdate
                (
                    recurringJobId: "beta job",
                    () => Console.WriteLine($"Recurring beta job觸發時間：{DateTime.Now}"),
                    cronExpression: "*/1 * * * *",
                    TimeZoneInfo.Local,
                    queue: "beta"
                );

alpha > beta > default

3、Hangfire Server設定。

a、設定Serve Name：用以識別個別機器。

一樣是在StartUp的ConfigureServices加入設定

    //Server Name
    options.ServerName = $"{Environment.MachineName}:NetCore";

接下來打開Hangfire儀表板Server的地方就會看到出現機器名稱。

b、設定Worker數量：Hangfire Server可以同時處理工作的數量。

這邊若一開始沒有特別給予特別的設定，預設值會是20。
而Hangfire官方建議配置數量為Hangfire Server的CPU數量乘以5。
Ex: 目前架設的Hangfire Server CPU數量為4核心，那就可以配置 4 * 5 = 20的Worker數量。

提醒：

Woker的設置數量並不是越多越好，必須考量目前架設Hangfire Server的CPU等級、數量，以及在同一台Server上的其他Hangfire服務。

設置地點同樣在StartUp的ConfigureServices。

//Worker
options.WorkerCount = 20;

也可以在儀錶板看到相關數量。

c、設定儀錶板路徑。

儀錶板設定的地方在StartUp的Configure。

            //Hangfire
            app.UseHangfireDashboard(
                 //dashboard path
                pathMatch: "/hangfire",
                options: new DashboardOptions
                {
                    Authorization = new List<IDashboardAuthorizationFilter>(),
                    IgnoreAntiforgeryToken = true
                });

d、設定首頁顯示的資訊。

一樣是在StartUp的ConfigureServices加入設定。

        services.AddHangfire(config =>
            {
                config.UseSqlServerStorage(
                    nameOrConnectionString: hangfireConnection,
                    options: new SqlServerStorageOptions
                    { 
                        SchemaName = "HangfireSample",
                        JobExpirationCheckInterval = TimeSpan.FromMinutes(value:60)
                    })
                //使用中的連線
                .UseDashboardMetric(SqlServerStorage.ActiveConnections)
                //連線數量
                .UseDashboardMetric(SqlServerStorage.TotalConnections)
                //Hangfire Server數量
                .UseDashboardMetric(DashboardMetrics.ServerCount)
                //Recurring Job數量
                .UseDashboardMetric(DashboardMetrics.RecurringJobCount)
                //排程數量
                .UseDashboardMetric(DashboardMetrics.ScheduledCount)
                //目前在Queue中的排程數量
                .UseDashboardMetric(DashboardMetrics.EnqueuedAndQueueCount)
                //處理中的排程數量
                .UseDashboardMetric(DashboardMetrics.ProcessingCount)
                //正在等待處理中的排程數量
                .UseDashboardMetric(DashboardMetrics.AwaitingCount)
                //已被刪除的排程數量
                .UseDashboardMetric(DashboardMetrics.DeletedCount)
                //執行失敗的排程數量
                .UseDashboardMetric(DashboardMetrics.FailedCount)
                //成功執行的排程數量
                .UseDashboardMetric(DashboardMetrics.SucceededCount);
            });

e、Job資料的保存期限設定。

  • 預設的SqlServerStorageOptions相關資料保存時間為一天。
  • Hangfire Server會每30分鐘去檢查是否有超過保存時間的資料如果超過保存時間就會予以刪除。

提醒：

太過頻繁檢查會使得Sql Server與 Hangfire Server附載過重。

可在StartUp的ConfigureServices加入設定。

            config.UseSqlServerStorage(
                    nameOrConnectionString: hangfireConnection,
                    options: new SqlServerStorageOptions
                    { 
                        SchemaName = "HangfireSample",
                        //Job資料的保存期限
                        JobExpirationCheckInterval = TimeSpan.FromMinutes(value:60)
                    })
