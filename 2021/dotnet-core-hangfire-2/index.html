dotnet Core HangFire - 2

Written by Sian on April 1, 2021 in 學習筆記 觀看次數:

前言

接下來試做HangFire。

本文

1、前置步驟。

一樣選擇Web應用程式。

接下來要選擇MVC的架構。

安裝以下三個Nuget套件。

  • Hangfire
  • Hangfire.AspNetCore
  • Newtonsoft.Json

在SSMS中localhost新建一個Hangfire DB。

2、設定。

把DB連線放在appsettings中加入connection string。

"ConnectionStrings": {
    "HangFire": "Server=localhost;Database=Hangfire;Trusted_Connection=True;"
  }

StartUp加入設定。

        public void ConfigureServices(IServiceCollection services)
        {
            services.AddControllersWithViews();

            //Hangfire
            var hangfireConnection = this.Configuration.GetConnectionString(name:"Hangfire");

            services.AddHangfire(config =>
            {
                config.UseSqlServerStorage(
                    nameOrConnectionString: hangfireConnection,
                    options: new SqlServerStorageOptions
                    { 
                        SchemaName = "HangfireSample",
                        JobExpirationCheckInterval = TimeSpan.FromMinutes(value:60)
                    });
            });

            services.AddHangfireServer();
        }
        
        public void Configure(IApplicationBuilder app, IWebHostEnvironment env)
        {
            if (env.IsDevelopment())
            {
                app.UseDeveloperExceptionPage();
            }
            else
            {
                app.UseExceptionHandler("/Home/Error");
                // The default HSTS value is 30 days. You may want to change this for production scenarios, see https://aka.ms/aspnetcore-hsts.
                app.UseHsts();
            }
            app.UseHttpsRedirection();
            app.UseStaticFiles();

            app.UseRouting();

            app.UseAuthorization();

            app.UseEndpoints(endpoints =>
            {
                endpoints.MapControllerRoute(
                    name: "default",
                    pattern: "{controller=Home}/{action=Index}/{id?}");
            });

            //Hangfire
            app.UseHangfireDashboard(
                pathMatch: "/hangfire",
                options: new DashboardOptions
                {
                    Authorization = new List<IDashboardAuthorizationFilter>(),
                    IgnoreAntiforgeryToken = true
                });
        }

接下來就可以啟動專案。
連線到Hangfire儀錶板，預設路徑會是https://{localhost啟動的port}/hangfire。

查看DB Hangfire的地方會發現已經生成了預設的Table。

這邊的TableName不可以隨意更換，如果要生成新的可以透過StatUp SchemaName修改。

3、測試。

在Controller加入排程。

        public IActionResult Index()
        {
            BackgroundJob.Enqueue(() => Console.WriteLine("Index頁面觸發：Fire and Foreget Schedule!"));
            return View();
        }

        public IActionResult Privacy()
        {
            BackgroundJob.Schedule(
                () => Console.WriteLine("Privacy頁面觸發：Delay Schedule!"),
                TimeSpan.FromMinutes(1));
            return View();
        }

        public IActionResult Contact()
        {
            var firstJob = BackgroundJob.Enqueue(
                () => Console.WriteLine("Contact頁面觸發：Continuations Job 1!)"));

            BackgroundJob.ContinueJobWith(
                firstJob,
                () => Console.WriteLine("Contact頁面觸發：Continuations Job 2!)"),
                JobContinuationOptions.OnlyOnSucceededState
                );
            return View();
        }

在StartUp加入Recurring job。

            //Recurring job
            RecurringJob.AddOrUpdate
                (
                    () => Console.WriteLine($"Recurring job觸發時間：{DateTime.Now}"),
                    cronExpression:"*/1 * * * *",
                    TimeZoneInfo.Local
                );
